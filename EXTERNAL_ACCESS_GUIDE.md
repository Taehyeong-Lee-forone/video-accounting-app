### オプション2：リモートデスクトップ

3. ローカル環境で操作

### オプション3：バックエンドも公開（推奨）
後述の「完全公開版セットアップ」を参照

## 🚀 完全公開版セットアップ（技術者向け）

### 方法1：ngrok有料プラン
```bash
# 2つのトンネルを同時に起動
ngrok start frontend backend --config ngrok.yml
```
- 月額$8で複数トンネル可能
- フロントエンド・バックエンド両方公開

### 方法2：Render.com（無料）
1. GitHubにコードをプッシュ
2. Render.comでバックエンドをデプロイ
3. 環境変数を更新
```
NEXT_PUBLIC_API_URL=https://your-backend.onrender.com
```

### 方法3：ローカルプロキシ設定
```bash
# ブラウザ拡張機能を使用
1. "Requestly" or "ModHeader"をインストール
2. localhost:5001 → 実際のバックエンドURLにリダイレクト設定
```

## 💡 よくある質問

### Q: なぜ「Visit Site」が表示される？
A: ngrokの無料プランの仕様です。セキュリティ警告ではありません。

### Q: いつまでアクセス可能？
A: ngrokセッションは8時間有効。その後は新しいURLが必要。

### Q: モバイルからアクセスできる？
A: はい、スマートフォンのブラウザからもアクセス可能です。

### Q: データは保存される？
A: 現在はローカルデータベースのため、セッション終了後は消えます。
